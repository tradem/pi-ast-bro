# pi-ast-bro

A [Pi](https://pi.dev) extension that integrates the [`ast-bro`](https://github.com/badlogic/ast-bro) CLI for AST-based code navigation, token-efficient context building, and pre-flight syntax checks.

## What it does

- **Token-saving reads**: Large source files (> 500 lines by default) are transparently replaced with an AST outline generated by `ast-bro map` when the agent calls `read` without `limit`/`offset`.
- **Raw fallback**: If the agent needs the complete raw source, it can call `read` again with explicit `limit` and `offset`. This is enforced at the tool level, not by configuration flags.
- **Pre-flight syntax checks**: After `edit` or `write`, the changed file is silently parsed with `ast-bro map`. If parsing fails, the tool result is marked as an error so the agent can fix the syntax immediately.
- **Dedicated AST tools**: Exposes `analyze_ast_impact`, `analyze_ast_map`, and `analyze_ast_search` as LLM-callable tools backed by `ast-bro`.
- **Interactive dashboard**: The `/ast` command opens a TUI overlay showing session stats, `ast-bro` availability, and live toggles for settings.

## Supported languages

Whatever `ast-bro` supports. By default the extension intercepts:

- `.rs` (Rust)
- `.cs` (C#)
- `.ts`, `.tsx` (TypeScript)
- `.py` (Python)
- `.dart` (Dart)

You can change, add, or remove extensions in the dashboard or in the settings file.

## Requirements

- [Pi](https://pi.dev) coding agent
- [`ast-bro`](https://github.com/badlogic/ast-bro) binary on your `PATH`
- Node.js >= 22

## Installation

### 1. Install `ast-bro`

Follow the upstream instructions, for example with cargo:

```bash
cargo install ast-bro
```

Make sure `ast-bro --version` works in the same shell where you start Pi. If Pi cannot find it, create a symlink to a directory that is always on `PATH`:

```bash
sudo ln -s "$(which ast-bro)" /usr/local/bin/ast-bro
```

### 2. Install the extension

#### Option A: Global extension (recommended for daily use)

```bash
mkdir -p ~/.pi/agent/extensions
ln -s /path/to/pi-ast-bro ~/.pi/agent/extensions/pi-ast-bro
```

Then start Pi normally from any project.

#### Option B: Project-local extension

Inside the project where you want to use it:

```bash
mkdir -p .pi/extensions
ln -s /path/to/pi-ast-bro/src/index.ts .pi/extensions/pi-ast-bro.ts
```

You will need to trust the project with `/trust` when Pi asks.

#### Option C: Temporary test from the repo

```bash
cd /path/to/pi-ast-bro
pi -e ./src/index.ts
```

### 3. Verify

Start Pi and run:

```text
/ast
```

You should see the dashboard with:

- `ast-bro status: available`
- Runtime stats
- Toggles for interceptors and threshold

## Usage

### Ask for high-level explanations

Prompts that only need structure and signatures work best with the AST summary:

```text
Erkl√§re mir den DD Core Bounded Context anhand der AST-Struktur und Signaturen.
```

### Force raw source

If you need the complete file content, ask the agent to read with explicit pagination. Any `read` call that includes `limit` and/or `offset` bypasses the interceptor automatically:

```text
Lies die Zeilen 1 bis 200 von src/main.rs im Original.
```

### Settings

Settings are stored per project at:

```text
.pi/plugins/ast-bro/settings.json
```

Example:

```json
{
  "enabled": true,
  "supportedExtensions": [".rs", ".cs", ".ts", ".tsx", ".py", ".dart"],
  "fileSizeThresholdLines": 500,
  "enablePreFlightSyntaxChecks": true
}
```

You can also edit these values interactively with `/ast`.

## Architecture

- `src/index.ts` ‚Äî Extension factory, startup lifecycle, and auto-installer prompt.
- `src/config.ts` ‚Äî Persistent settings manager using Typebox schemas.
- `src/state.ts` ‚Äî Session-level stats (bytes saved, reads intercepted, pre-flight errors).
- `src/interceptors.ts` ‚Äî `tool_call` interceptors for `read`/`view_file` and `tool_result` middleware for `edit`/`write`.
- `src/tools.ts` ‚Äî `analyze_ast_impact`, `analyze_ast_map`, `analyze_ast_search` tool definitions.
- `src/tui.ts` ‚Äî `/ast` dashboard component.
- `src/utils.ts` ‚Äî Safe path validation, `ast-bro` execution helpers, availability checks.

## Security notes

- `ast-bro` is invoked via `spawnSync` with argument arrays, never shell strings, to prevent command injection.
- File paths are validated before being passed to the OS.
- If `ast-bro` crashes, hangs, or is missing, the extension falls back to Pi‚Äôs default tool behavior.
